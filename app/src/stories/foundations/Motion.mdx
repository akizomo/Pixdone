# Motion Tokens

Motion is **fast and clear by default**, and **expressive only when it supports feedback** (e.g. reward, smash). Standard interactions stay within 120–240ms; reward effects may be longer but should still feel snappy. Respect `prefers-reduced-motion: reduce` via `tokens.css` duration overrides.

---

## Principles

- **Fast feedback** – Use instant/fast/base (80–180ms) for hover, focus, toggles, and button press so the UI feels responsive.
- **Expressive reward animation** – Use reward (700ms) and playful easing for completion and celebration; keep it snappy, not slow.
- **Avoid long UI animations** – Do not use long durations for routine open/close or layout; reserve longer motion for reward/special moments.

## CSS variables

`--pxd-motion-fast`, `--pxd-motion-base`, `--pxd-easing-standard`, `--pxd-easing-playful`, `--pxd-scale-press`, `--pxd-scale-reward-pop`, `--pxd-opacity-disabled`. Reduced motion overrides in `tokens.css` set duration variables to 0ms when `prefers-reduced-motion: reduce`.
