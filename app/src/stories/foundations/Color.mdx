# Color Tokens

Pixdone uses a **Primitive → Semantic** color architecture.

- **Primitive** tokens (`--pxd-gray-*`, `--pxd-ink-*`, `--pxd-purple-*` …) define the raw palette.
- **Semantic** tokens (`--pxd-color-surface-page`, `--pxd-color-text-primary` …) assign UI meaning and are overridden per color mode.
- **Components** must only reference semantic tokens — never primitives directly.

The table below shows how each semantic token resolves in **Light** and **Dark** mode.

## CSS変数の使い方

コンポーネントでは `--pxd-` プレフィックスのセマンティックトークンを使用する。プリミティブトークン (`--pxd-gray-*`, `--pxd-ink-*`) は直接参照しない。

```css
/* ✅ 正しい — セマンティックトークンを参照 */
color: var(--pxd-color-text-primary);
background: var(--pxd-color-surface-page);

/* ❌ 誤り — プリミティブを直接参照 */
color: var(--pxd-gray-900);
background: var(--pxd-ink-950);
```
