# Sound

Pixdone では **すべての操作** に選択音・フィードバック音を割り振り、レトロで遊びのある体験にしています。**ルールとトークン割り当ては vanilla（public/script.js および .cursor/rules/pixdone-sounds.mdc）と同一**です。

---

## 原則（vanilla 準拠）

1. **すべての操作に音を割り当てる**  
   タスク追加・編集・削除・完了・キャンセル・ボタンクリック・ドラッグ並び替えなど、ユーザーが行う操作には必ず `comicEffects.playSound(...)` / `playSound(key)` を付ける。

2. **重要度別サウンド割り当て（vanilla ルール）**  
   - **taskAdd** … データ追加（タスク/リスト/サブタスク追加、確定）  
   - **taskDelete** … データ削除（タスク/リスト削除）  
   - **taskComplete** … 成功・完了（ログイン/ログアウト成功、パスワードリセット成功）  
   - **taskEdit** … 編集・モード開始（タスク編集を開く、サブタスク編集）  
   - **taskCancel** … キャンセル・閉じる（モーダルを閉じる、キャンセル）  
   - **buttonClick** … 選択・トグル（タブ、チップ、サウンドトグル）  
   - **subtaskComplete** … サブタスク完了・コンボビープ（vanilla: picoSound.playSubtaskCompleteSound）

3. **同時再生は避ける**  
   1 度に 1 つの UI サウンド。連打にはデバウンスする。

4. **ミュートを常に提供する**  
   ユーザーが音をオフにできるようにする。`pixdone-sound-enabled`（localStorage）と ComicEffectsManager で同期する。

5. **重要な情報を音だけに頼らない**  
   エラーや成功など、意味のある情報は視覚（テキスト・アイコン・状態）でも伝える。音は補助。

## 使用例（コード）

```ts
import { playSound } from '../services/sound';
import { sound } from '../design-system';

// タスク追加確定時
playSound(sound.taskAdd.key);  // 'taskAdd'

// モーダルを閉じた時
playSound(sound.taskCancel.key);  // 'taskCancel'

// 言語チップ・タブなど
playSound(sound.buttonClick.key);  // 'buttonClick'
```

`services/sound.ts` の `playSound(key: SoundKey)` に、上記の `sound.*.key` を渡す。ミュート時は内部で再生しない。

## 役割別の対応表（実装チェック用）

| 役割 | トークン | 主な使用箇所 |
|------|----------|--------------|
| データ追加 | taskAdd | タスク保存、リスト作成、サブタスク追加確定 |
| 編集・確定 | taskEdit | 編集シートを開く、タスク/リスト保存、リスト名変更 |
| データ削除 | taskDelete | 削除確認ダイアログで「削除する」 |
| キャンセル | taskCancel | シート閉じる、キャンセルボタン、モーダル閉じる |
| 完了・成功 | taskComplete | タスク完了、ログイン/ログアウト成功 |
| 選択・トグル | buttonClick | タブ、チップ、トグル、日付ボタン、その他汎用ボタン |
| サブタスク完了 | subtaskComplete | サブタスクのチェックをオンにしたとき（picoSound） |

## コンポーネント別サウンド割り当て（vanilla と同一）

各 UI コンポーネントで鳴らす音。vanilla の割り当てに合わせる。

| コンポーネント / 操作 | サウンド |
|------------------------|----------|
| **Button**（primary / secondary / signup / danger） | buttonClick（押下時） |
| **IconButton** | buttonClick |
| **Chip**（選択・タブ・言語など） | buttonClick（選択時）、taskCancel（onRemove 時） |
| **ModalDialog**（オーバーレイ閉じる・ESC） | taskCancel |
| **BottomSheet**（閉じるボタン） | taskCancel |
| **TaskForm** | taskAdd（保存・サブタスク追加）、taskCancel（キャンセル・サブタスク削除）、buttonClick（今日/明日/日付/繰り返し）、subtaskComplete（サブタスクチェック） |
| **ListHeader**（メニュー開閉・名前変更・削除） | buttonClick |
| **ListTabs**（タブ切替・リスト追加） | buttonClick |
| **ListModal**（キャンセル・オーバーレイ） | taskCancel；確定時は taskAdd / taskEdit / taskDelete（モードに応じ） |
| **AuthModal**（チップ・閉じる・パスワード表示・Forgot/Back） | buttonClick（チップ等）、taskCancel（閉じる）、taskComplete（ログイン/サインアップ/リセット成功） |
| **ThemeSelector**（テーマ選択） | buttonClick |
| **TutorialPanel**（サインアップボタン） | buttonClick |
| **削除確認モーダル**（キャンセル） | taskCancel；（削除実行）taskDelete |
| **完了セクション展開トグル** | buttonClick |
| **キーボード左右（リストタブ）** | buttonClick |
| **サウンドトグル（オフ→オン時）** | buttonClick |
| **ログアウト** | taskComplete |

## ミニゲーム・拡張（任意）

Perfect Timing などのミニゲームでは、次のようなキーを別途実装で扱うことがある。

- `perfectTimingMiss` / `Good` / `Great` / `Perfect`  
- `perfectTimingGaugeTick`  

これらは必要に応じて `sound.tokens.ts` や実装側で拡張する。

---

## アクセシビリティ

- **ミュート** – 常にオフにできること。設定またはヘッダードロップダウンのサウンドトグルで提供。
- **音だけに依存しない** – エラー・成功・状態変化は、テキストやアイコンでも伝える。
- **自動再生** – ユーザー操作に紐づけて鳴らし、意図しない自動再生は避ける。
