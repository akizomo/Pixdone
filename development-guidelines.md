# 開発ガイドライン (Development Guidelines)

## 1. コンポーネント設計原則

### 1.1 単一責任原則
- 各コンポーネントは1つの明確な責任を持つ
- 複数の機能を持つ場合は、サブコンポーネントに分割する

### 1.2 再利用可能性
- デスクトップ・モバイルで共通のロジックを使用
- 設定により動作を変更可能な設計

### 1.3 状態管理
- ローカル状態：コンポーネント内部の状態
- グローバル状態：複数コンポーネント間で共有する状態
- 状態の変更は必ずStateManagerを通す

## 2. 命名規則

### 2.1 コンポーネント名
- PascalCase: `MobileModal`, `TaskList`
- 機能を表す明確な名前

### 2.2 メソッド名
- camelCase: `showModal`, `addTask`
- 動詞で始まる

### 2.3 イベント名
- kebab-case: `task-added`, `modal-closed`
- 過去形で命名

### 2.4 CSS クラス名
- kebab-case: `mobile-modal`, `task-item`
- BEM記法を推奨: `component__element--modifier`

## 3. イベント管理

### 3.1 イベントリスナーの追加
```javascript
// 良い例：EventManagerを使用
component.on('click', handler);

// 悪い例：直接追加
element.addEventListener('click', handler);
```

### 3.2 イベントの削除
- コンポーネント破棄時に自動削除される
- 手動削除する場合は返されるIDを使用

### 3.3 カスタムイベント
```javascript
// イベントの発行
component.emit('task-added', taskData);

// イベントの購読
component.on('task-added', (data) => {
    // 処理
});
```

## 4. 開発ワークフロー

### 4.1 新機能開発の手順
1. 設計検討（コンポーネント分割、状態管理）
2. コンポーネント作成・登録
3. テスト作成・実行
4. 統合テスト
5. ドキュメント更新

### 4.2 バグ修正の手順
1. 現象の再現
2. デバッグツールでの分析
3. 原因特定
4. 修正実装
5. テスト実行
6. 再発防止策の検討

### 4.3 コードレビューのポイント
- コンポーネント設計の妥当性
- 状態管理の適切性
- イベント管理の正確性
- テストの十分性

## 5. テストガイドライン

### 5.1 テストの種類
- 単体テスト：個別コンポーネントの動作
- 統合テスト：コンポーネント間の連携
- E2Eテスト：ユーザー操作シナリオ

### 5.2 テストケース
```javascript
// コンポーネントの基本テスト
{
    name: 'Element exists',
    test: () => component.element !== null
},
{
    name: 'Can show/hide',
    test: () => {
        component.methods.show();
        const isVisible = component.state.isOpen;
        component.methods.hide();
        return isVisible && !component.state.isOpen;
    }
}
```

### 5.3 テストの実行
```javascript
// 個別コンポーネントのテスト
dev.test('mobileModal');

// 全てのテストを実行
dev.testAll();
```

## 6. デバッグ手順

### 6.1 基本的なデバッグ
```javascript
// コンポーネントの状態確認
dev.debug('componentName');

// 全コンポーネントの一覧
dev.components();

// 現在の状態確認
dev.state();
```

### 6.2 エラーが発生した場合
1. コンソールでエラーメッセージを確認
2. `dev.debug()`でコンポーネントの状態を確認
3. `dev.autofix()`で自動修復を試行
4. 必要に応じて`dev.reset()`で初期化

## 7. パフォーマンス最適化

### 7.1 イベントリスナーの最適化
- 不要なイベントリスナーの除去
- イベントデリゲーションの活用
- 高頻度イベントの間引き

### 7.2 状態管理の最適化
- 不要な状態変更の削減
- 状態の正規化
- メモ化の活用

### 7.3 パフォーマンス監視
```javascript
// パフォーマンス報告
dev.performance();
```

## 8. トラブルシューティング

### 8.1 よくある問題と解決策

#### 問題：コンポーネントが表示されない
- 解決策：`dev.debug()`で要素の存在を確認
- 原因：セレクターの間違い、CSSの問題

#### 問題：イベントが発火しない
- 解決策：`dev.events()`でイベントリスナーを確認
- 原因：イベントリスナーの重複登録、要素の取得失敗

#### 問題：状態が更新されない
- 解決策：`dev.state()`で状態を確認
- 原因：直接的な状態変更、StateManagerの未使用

### 8.2 緊急時の対処
```javascript
// 全てをリセット
dev.reset();

// 自動修復を試行
dev.autofix();
```
